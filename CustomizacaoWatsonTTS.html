Customização de palavras no Watson Text to Speech¶

Se voc j testou alguma soluo de mercado para a sintetizao de udio a partir de texto (text to speech) ento voc provavelmente j se deparou com alguma palavra que soou errado, isso ocorre porque as soluo de fala para texto (TTS) so treinadas considerando o portugus comum que encontramos no dicionrio.

No entanto muito comum encontrarmos siglas, termos especficos ou at mesmo a utilizao de palavras estrangeiras dentro de uma empresa nas nossas respostas. Como podemos adaptar nosso sistema cognitivo para que possa suportar tais especializaes?

Um mtodo seria escrever a palavra errada em nossas respostas para que ento ela fosse vocalizada corretamente nos sistemas que utilizam udio, mas essa abordagem nos tira a flexibilidade de poder treinar um mesmo corpus e permitir a utilizao tanto por texto quanto por voz.

Nesse tutorial vamos apresentar como customizar o Watson Text to Speech de modo que ele fale as suas palavras corretamente em portugus.

Teste de sintetização¶

Voc j tentou reproduzir uma palavra em ingls quando configurou o seu text-to-speech em portugus? Vamos fazer um teste com a palavra "text-to-speech".

Precisamos passar os seguintes parmetros para o Watson:

  • text a frase/palavra a ser sintetizada.
  • accept o tipo de arquivo a ser retornado (formato do udio).
  • voice a voz que utilizaremos, para portugus devemos especificar o valor pt-BR_IsabelaVoice

O mesmo comando em cURL:

curl -X GET -u "{username}":"{password}" --output exemplo_audio_1.wav "https://stream.watsonplatform.net/text-to-speech/api/v1/synthesize?accept=audio/wav&text=text-to-speech&voice=pt-BR_IsabelaVoice"

Cria um modelo novo do seu modelo de customização¶

Para corrigir a pronncia do nosso TTS devemos criar um novo modelo de customizao e para isso devemos chamar o mtodo create_voice_model passando os seguintes parmetros:

  • name o nome do modelo a ser criado, voc pode especificar o nome que quiser.
  • language dever ser pt-BR para que o modelo seja utilizado considerando o portugus do Brasil.
  • description opicional e permite que voc coloque mais informaes sobre o modelo.

A resposta dessa chamada ser um json no qual devemos salvar o valor do campo customization_id que ser utilizado depois.

O mesmo comando em cURL:

curl -X POST -u "{username}":"{password}" --header "Content-Type: application/json" --data "{\"name\":\"medium-custom_2\", \"language\":\"pt-BR\", \"description\":\"\"}" "https://stream.watsonplatform.net/text-to-speech/api/v1/customizations"

In [5]:
obj_customization = text_to_speech.create_voice_model(name='medium-custom_2',language="pt-BR",description="")
customization_id = obj_customization["customization_id"]

Adiciona palavras ao modelo¶

Depois que criamos o modelo de customizao basta adicionar as palavras e suas respectivas transcries ao modelo, para isso podemos fazer palavra por palavra ou ento por "batch" (no faremos aqui mas voc apenas precisaria montar um arquivo json e submeter o mesmo para o servio do TTS usando o mtodo add_words).

No caso de palavra por palavra devemos chamar o mtodo add_word com o seguintes parmetros:

  • customization_id o id do seu modelo de customizao.
  • word a palavra que hoje possui uma sintetizao problemtica.
  • translation o como a palavra deveria soar. [1]

[1] A parte de translation voc pode realizar de duas formas diferentes:

  • Sounds-like: como a palavra deveria soar? Aqui fazemos a traduo utilizando similaridade.
  • Phonetic: trabalhamos em como descrever a pronncia de uma palavra atravs de fonemas, para isso voc provavelmente precisar de um linguista mas tambm poder atingir resultados mais corretos.

Aqui utilizaremos apenas a customizao via sounds-like, caso voc queira realizar via fonemas basta acessar este link: https://console.bluemix.net/docs/services/text-to-speech/custom-intro.html#phonetic.

O mesmo comando em cURL:

curl -X PUT -u "{username}":"{password}" --header "Content-Type: application/json" --data "{\"translation\":\"tcsti tchu spit\"}" "https://stream.watsonplatform.net/text-to-speech/api/v1/customizations/6af96480-e8dd-4087-aa27-9e07d5b075c9/words/text-to-speech"

In [7]:
# Exemplo para adicionar apenas uma palavra
print(text_to_speech.add_word(customization_id,word='text-to-speech',translation='tcsti tchu spit'))

Conclusão¶

Vimos aqui que apenas com duas chamadas (create_voice_model e add_word) ao Text to Speech da IBM podemos customiz-lo para o portugus do Brasil, essa possibilidade de extrema importncia para que possamos criar nosso prprio vocabulrio e a vocalizao de tais palavras da maneira correta.
